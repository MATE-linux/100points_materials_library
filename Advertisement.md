# Объявление: Начинается набор в корреспонденты сайта  
1. Требования:  
  - [ ] Быть записанным на полуторагодовой курс в 100 бальном по любому предмету
  - [ ] Быть готовым задерживаться на 10-15 минут после урока для того, чтобы скачать и опубликрвать конспекты
  - [ ] Иметь Telegram или другой способ для связи
2. Технические требования:  
  - [ ] Иметь ПК с доступом в интернет
  - [ ] Иметь доступ к командной строке / Github desktop при наличии аккаунта github
  - [ ] Установить Node.js(Версия 18 или новее)
  - [ ] Установить Obsidian для редактирования `.md` файлов, из которых генерируется сайт / другой удобный редкатор markdown
  - [ ] Скачать исходники сайта по [ссылке](https://github.com/MATE-linux/100points_library_site/tree/v4 ) (кнопка code > Download zip, затем распаковать скачанный архив, либо скачать другим удобным способом). Файлы, которые вы должны изменять, вы можете видеть в папке `content`
3. Приветствуются, но не обязательны (по вашему желанию могу добавить в организации  и группы):  
  - [ ] Аккаунт на [github](https://github.com)
  - [ ] Аккаунт на [vercel](https://vercel.app)
  - [ ] Аккаунт Matrix  
Технический процесс:    
    В первый раз:  
        1. Скачать исходники сайта  
        2. Открыть папку `content` в Obsidian, создать файлы согласно структуре:    
``` text
            project/
            ├── content/
            │   └── Math 10 class/ (папка предмета)
            │       └── ... (файлы конспектов согласно шаблону из Lesson Template.md)
                ├── resourses/ 
                    └── Math_10_class/ (ресурсы предмета - например pdf)
                │       └── Block_1/ (ресурсы за блок 1)
                │           └── теория_треугольники_зантие_2.pdf
                └── templates/
                |   └── Lesson Template.md
```  

    В последующие разы:  
    1. Сохранить конспект в папку `resourses`  
    2. Создать файл конспекта в папке предмета  
    3. Открыть папку всего проекта в терминале и выполнить команду `npx quartz build --serve`, выйдет ссылка, пройти по ней и проверить изменения, затем в командной строке нажать `Ctrl + C`
    4. Выполнить в папке всего проекта в командной строке `npx quartz sync`, это отправит изменения на сайт, примерно спустя 5 минут сайт обновится и вы увидите их там  
Откликнуться на предложение можно в комментариях под постом: https://t.me/points_library_update_bot/14
